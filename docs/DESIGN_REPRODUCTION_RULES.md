# DESIGN_REPRODUCTION_RULES.md — なごとしゃ モック再現専用ルール

---

## 基本方針

**モック画像は「参考」ではなく「正解」。**

「雰囲気が近ければOK」は禁止。
「モックの構成・余白・色・写真比率・文字サイズを正確に再現する」が目標。

---

## 絶対禁止

- 雰囲気だけの再現（「それっぽければOK」は禁止）
- CSSシルエットで写真の代替をする
- グラデーションで写真の代替をする
- Unsplash・Pixabay等の汎用写真を無断使用する（ライセンス確認必須）
- 外部画像URLをコードに直書きする（許可なし）
- 素材がないまま「後で差し替える前提」で雰囲気実装を進める
- モックにない要素を勝手に追加する
- モックにある要素を「難しいから」と省略する

---

## 素材不足時のルール

**素材（写真・アイコン・ロゴ等）がない場合は、代替実装をしてはいけない。**

以下の手順で止まる：

1. 何の素材が不足しているかを報告する
2. 代替案を「候補として提示」する（実装しない）
3. ユーザーの判断を待つ
4. 承認を得た方法のみ実装する

```
報告例：
「ヒーロー背景に使う名古屋の街並み写真が不足しています。
CSSグラデーションで代替することは禁止ルールに該当するため、
実装を停止します。以下の素材候補を確認してください。」
```

## 再現チェック項目（セクションごとに確認）

- [ ] セクション順序がモックと一致しているか
- [ ] 余白（上下左右のpadding/margin）がモックと近いか
- [ ] 背景色がモックと一致しているか
- [ ] 文字サイズ・太さがモックと近いか
- [ ] 見出しの強さ（font-weight・サイズ）がモックと近いか
- [ ] カード形状（角丸・影・枠線）がモックと近いか
- [ ] 写真比率（縦横比）がモックと近いか
- [ ] 検索バーの位置・サイズがモックと近いか
- [ ] CTAの位置・色・形がモックと近いか
- [ ] スマホ（390px）での表示密度がモックと近いか
- [ ] 下部ナビ（タブバー）がモックと近いか
- [ ] フッターのデザイン・配色がモックと近いか

---

## 再現度の判定

各セクションの再現度を % で見積もる。

| 再現度 | 判定 | 対応 |
|---|---|---|
| 90%以上 | 合格 | コミット可能 |
| 70〜89% | 要修正 | 差分を報告して修正を続ける |
| 70%未満 | 不合格 | 作業を止めて再確認 |

**90%未満ではコミットしない。**

---

## モック再現タスクの完了条件

以下を全て満たすまで「完了」とは言わない。

1. 対象セクションがモックと90%以上一致している
2. `npm run build` が成功している
3. スマホ390px / 430pxで横スクロールが出ていない
4. ユーザーがスクショを確認して承認している
5. `git diff --cached --name-only` で対象ファイルのみが staging にある
