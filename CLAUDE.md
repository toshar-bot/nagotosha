## 絶対禁止事項

- `git add .` は絶対禁止。addは必ず個別ファイル指定
- `.env` は読まない・表示しない・変更しない
- 認証情報・パスワード・アプリケーションパスワードは一切表示しない
- WordPress投稿・更新・削除・publish は禁止
- ユーザーから明示的に許可されていないコミットは行わない

## Claude固有のルール

### 差分を小さく保つ
- 整理・調査・仕様化は得意だが、実装時は差分を最小限にする
- 一度に複数のファイル・複数のセクションを大幅変更しない
- 頼まれていない改善・リファクタ・整形は行わない

### デザイン判断
- 「いい感じ」ではなく「モック準拠」で動く
- 勝手なデザイン解釈・勝手な補完は禁止
- モックと異なる実装をした場合は、その理由を明示する
- 素材不足・情報不足の場合は止まって報告する

### 作業前宣言（必須）
コード編集を始める前に必ず以下を明示する。

```
触るファイル: [ファイル名]
触らないファイル: [ファイル名]
目的: [何をするか]
完了条件: [何をもって完了とするか]
```

### 途中の方向転換禁止
- 作業中に別提案・別アプローチへの誘導をしない
- 「こっちの方がいい」という勝手な判断で変更先を広げない
- 不明点はその場で確認する

### モック再現タスク
- モック再現タスクでは、素材不足をCSS・グラデーション・シルエット・汎用画像で勝手に代替してはいけない
- 素材がない場合は作業を止め、必要素材を報告する
- 90%未満の再現度ではコミットしない

### build・表示確認
- build成功は完了条件ではない
- モックとの見た目比較、スマホ表示（390px / 430px）、横スクロールなし、ユーザー承認まで完了扱いしない

### コミット
- コミットはユーザー承認後のみ実施
- `git diff --cached --name-only` で対象ファイルを必ず確認
- 対象外ファイルが staging に入っていたら即停止・報告

## 報告フォーマット

作業後は必ず以下を報告する。

1. 変更したファイル一覧
2. 変更内容の要約
3. `npm run build` の結果
4. `git status --short`
5. `git add` / `commit` をまだしていないこと
6. 認証情報を表示していないこと
